● Project Overview: Spendly — Personal Expense Tracker

  This is a Flask-based web application for tracking personal expenses. It uses
  the MVT (Model-View-Template) architectural pattern, which is Flask's variant
  of MVC.

  ---
  Project Structure

  expense-tracker/
  ├── app.py              # Main Flask application (routes + logic)
  ├── database/
  │   └── db.py           # Database setup placeholder (Step 1)
  ├── static/
  │   ├── css/style.css   # All styling (warm earthy palette)
  │   └── js/main.js      # Frontend JS (placeholder)
  └── templates/
      ├── base.html       # Base layout with navbar/footer
      ├── landing.html    # Marketing homepage
      ├── login.html      # Sign in page
      └── register.html   # Account creation page

  ---
  How Flask is Used

  1. Application Factory & Routing (app.py)
  from flask import Flask
  app = Flask(__name__)
  - Flask(__name__) creates the WSGI application instance
  - @app.route("/") maps URLs to Python functions (view functions)
  - render_template() renders HTML templates with Jinja2

  2. Template Engine (Jinja2)
  Templates use Jinja2 syntax:
  - {% extends "base.html" %} — inheritance
  - {% block content %}{% endblock %} — section override
  - {{ url_for('register') }} — URL generation (generates /register)
  - {{ variable }} — template interpolation

  3. URL Routing
  @app.route("/register")     # GET /register
  @app.route("/login")         # GET /login
  @app.route("/logout")        # Placeholder
  @app.route("/profile")       # Placeholder
  @app.route("/expenses/add")  # Placeholder
  Dynamic routes like /expenses/<int:id>/edit capture path parameters.

  4. Static Files
  <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') 
  }}">
  The url_for() helper generates correct URLs to static assets.

  5. Form Handling
  Login/register forms POST to /login and /register routes (to be implemented in
   later steps).

  ---
  Why Flask?

  Flask is ideal here because:
  - Minimal setup — only 4 dependencies in requirements.txt
  - Flexible — no mandatory ORM, database, or templating constraints
  - Python-native — works seamlessly with Python's sqlite3
  - Good for learning — students see exactly how each component works
  - Scalable — can add Flask-SQLAlchemy, Flask-Login, Flask-Migrate as needed
